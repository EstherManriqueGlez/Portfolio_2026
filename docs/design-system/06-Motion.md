# Motion

## Purpose

Motion improves the user experience by providing feedback, guiding attention, and creating a refined interaction feel.

---

## Principles

- Motion should have a clear purpose.
- Animations should feel natural and subtle.
- Avoid unnecessary movement.
- Respect user preferences.
- Performance always comes first.

---

## Motion Categories

### Micro Interactions

Small feedback animations:

- Hover states
- Button interactions
- Focus states

### Transitions

Changes between states:

- Visibility
- Position
- Scale
- Color

### Entrance Animations

Used for:

- Hero elements
- Sections
- Content reveal

---

## Timing

Animations should use consistent durations:

- Fast: immediate feedback
- Medium: standard transitions
- Slow: emphasis and storytelling

---

## Accessibility

Respect `prefers-reduced-motion` to provide a comfortable experience for all users.
