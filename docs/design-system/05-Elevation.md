# Elevation

## Purpose

Elevation creates visual hierarchy by defining how components are layered and perceived. It should be subtle, consistent, and never distracting.

---

## Principles

- Prefer depth through spacing before shadows.
- Use shadows sparingly.
- Elevation should communicate interaction.
- Maintain consistent layering across components.

---

## Elevation Levels

### Level 0

Default surfaces without shadow.

### Level 1

Subtle separation for cards and panels.

### Level 2

Interactive components such as dropdowns and popovers.

### Level 3

High-priority overlays such as dialogs and modals.

---

## Borders

Borders complement elevation and should remain subtle, using neutral colors whenever possible.
